WORKSPACES: Before starting: check if project/workspace exists with list_workspaces or get_workspaces. Create new workspace per project to avoid conflicts. Use descriptive names (e.g. project-name, ai-chat-interface). Reuse existing workspace if continuing same project. All paths are workspace-relative; use same workspace name for all tools in one session. Clean up old workspaces regularly with delete_workspace when done. Exception: Never delete workspaces that are cloned from a workspace repo (e.g. document-creator, file-converter, data-analysis, linux-expert); these persist across tasks. Each workspace is isolated; changes persist between handoffs.
